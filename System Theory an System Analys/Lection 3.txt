Существуют различные классификации связей в онтологических отношениях:
	1.	Связь типа Генеология (предок - потомок)
	2.	Таксономия (структура классификации животного мира - например, 5 царств)
	3.	Партономия (свяь типа a part of - APO)
	4.	Топологическая связь (пути различных взаимодействий)
		Причинно-Следственные связи
		Временные свящи
		Сущность поддерживает другую сущность
		Пространственные связи
		Социальные связи:
			Связь Владеет
			Связь Проживает
			Связь Цель-Результат

Стандарт IDEF5 не ограничивает число связей, которые можно заводить в онтологии.
Однако максимально стремимся вначале задавать связи иерархические, а потом, как исключение, топологические

Связь типа Управление приходит из пространства состояний:
	x' = f(x, t); x(0) = c
	Есть система, которая в начале времени имела значение с.
	Производная в уравнении означает, что система имеет память - состояние зависит от предыдущего состояния.
	В этом уравнении показан характер поведения системы, но не сказано, как на него влиять.
	Для этого в наборе x в явном виде выделяется компонента u: x'=f(x,u,t)
	u - компонента времени. u - это и есть управление
	Мы явно выделяем те компоненты, на которые мы можем влиять и которые параметризуют остальные компоненты состояния
	На что обычно влияют?
		1.	На выходной параметр системы (силовое управление)
			Бежит толпа, что делать? Ставим турникет. Едит машина - жмем в тормоза.
		2.	На внутренние параметры системы (параметрическое управление)
			Едет машина - меньше бензина, коробка передач.
		3.	Изменяем структуру системы (структурное управление)

	Какую информацию учитываем при управлении?
		1.	Никакую (программное управление)
			Распостранено в системах, где внешняя среда оказывает несущественное влияние на траекторию системы
		2.	Учитываем входную информацию (управление по возмущению)
			Ставится обратная связь на вход. Пример - система кондиционирования
		3.	Учитываем выходную информацию (управление по отклонению)
			Телескоп надо держать наведенным куда-то. Если уедет - изображение уедет.
		4.	Учитываем информацию о текущем состоянии (управление по состоянию)



Процессы.

	Определения процессов.
		Процесс - последовательность состояний естественных или искусственных систем, связность стадий их изменений и развития.
			1.	Направленность в изменении объектов
			2.	Временной характер (было, стало. Причина, следствие.)
		Технологический процесс - часть общепроизводственного процесса, содержащего целенаправленные действия по изменению или определению состояния предмета труда.
			Предметы труда - заготовки  изделия. ГОСТ 3.1109-82
			Технологический процесс разделяется на элементарные технологические процессы (или на технологические операции)
			Технологическая операция - наименьшая часть ТП, обладающая всеми его свойствами, т.е. такое ТП, дальнейшая декомпозиция которого приводит к потере признако, характерных для метода, положенного в основу данной технологии.
		Computer Science процесс - выполняющаяся программа + все ее компоненты (адресное пространство, регистры, стэк, файлы...)
		Стандарт ISO 9000-2000 (Бизнес-информатика): Процесс - совокупность взаимосвязанных и взаимодействующих видов действий, преобразующих входы в выходы

		Процесс - это целенаправленная активность!
		Причем активность в искуственных (!) системах.
		Следовательно, здесь могут быть разные целеполагания. В частности, к процессу может быть отнесено собственно производство выхода и управление производством выхода.

		Управление на основе бизнес процессов лежит в основе управления большинства бизнес-технологий.
		Главный принцип процессного подхода в управлении состоит в следующем: один процесс - одно подразделение - один бюджет - один владелец процесса.

		Бизнес процесс может включать как формальные, так и неформальные взаимодействия между участниками.
		Бизнес процесс может состоять из автоматизированных и ручных активностей.

		Активность - Activity - описание части работы, которая формирует один логический шаг процесса.
		Активность - самая малая единицы работы, которая поддерживается движком управления работы.

		Бизнес процесс - набор активностей, организованный относительно некоторой цели.
		Бизнес процесс по признаку цели классифицируется:
			1.	Группируем действия по предмету действия (по схожим функциям)
				Делим работников подразделения по выполняемым функциям (операторы, уборщики и т.д.)
			2.	По результату действия (по продукту)
				По цели заказчика и работы для него:
					1 группа. Процессы типа Потребитель-Поставщик
					2 группа. Инженерные процессы. Процессы непосредственно специфицирующие рализующие продукт.
					3 группа. Вспомогательные процессы.
					4 группа. Управленчиские - процессы, содержащие общие действия по управльльзлению проектом любого типа.
					5 группа. Организационные - определенные бизнес целей организации и разработка всех других процессов.
						Это и есть маркетинг.		
			3.	По добавленной ценности для клиета.
				Получение максимальной выгоды (что выгоднее - написать новое, или мусолить старое?)

		А какая из моделей лучше?
		Вопрос не имеет однозначного ответа.
		Гораздо более органичным является матричный подход. Когда мы имеем 2 координаты.
		Координата подсистемы (работы сформированной по предмету деятельности) и координата бизнес процессов (последовательностей работ)

		Описания:
			1. Процессное описание (по процессам)
			2. Функциональное описание (по активностям)
			3. Проектное описание (по матрице - точнее по ее кусочкам)

		Референтные модели в определении бизнес-процессов.
			Типичный поход к типизации - наличие референтных (=опорных, базовых) моделей.
			Референтная модель - обобщенная модель, на базе которой можно составить свою модель для собственного бизнеса.
			Варианты:
				1. Референтная модель цепи поставок:
					1.	Планирование
					2.	Получение продуктов
					3.	Производство продуктов
					4.	Поставка продуктов
					5.	Возврат
				2. Референтная модель целиком:
					1.	Исполнение услуги
					2.	Обеспечение услуги
					3.	Биллинг (оплата)
				3. Модель ITIL (Библиотека типовых практив управления производством)
				4. Американский центр производительности и качества имеет перечень - 250 процессов.
