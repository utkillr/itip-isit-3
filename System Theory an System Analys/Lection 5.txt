Смежность графов

	Рассмотрим простой граф без петель
	Маршрут в таком графе - это последовательность смежных ребер
	Маршрут в мультиграфе - последовательность из вершин и ребер, где любые два соседних элемента инцедентны
	Цепь - маршрут, все ребра которого различны
	В орграфе цепь называется путем (путь)
	Цикл - замкнутая цепь. В орграфе цикл - это контур

	Граф называется связным, если любые две его вершины связаны маршрутом
	Компонент связности - это подграф, образованный на множестве всех вершин, которые можно соединить произвоьным маршрутом.
	Связный граф состоит из единственной компоненты связности.
	Если у графа более одной компоненты связности, то он не связный.

	В типах связности выделяются отдельные специфические циклы.
	Эйлеров цикл - цикл, где содержатся все ребра графа, причем каждое ребро встречается только один раз.
	Граф содержит Эйлеров цикл, если он связен и все локальные степени его вершин четные.
	Примеры:
		Из какого минимального числа кусков проволоки можно собрать каркас куба? (ответ 4)
		Как создать план маршрутов, чтобы посетитель мог пройти по каждому месту 1 раз?
		Кеннингбергские мосты.

	В реальном мире Эйлеровы графы встречаются редко.

	Цикл Гамельтона проходит через каждую вершину графа 1 раз.
	Общего критерия для Гамельтонова графа нет.
	Задача нахождения Гамельтонова графа тяжелая, но встречается часто.
	Примеры:
		Можно ли сходить конем так, чтобы посетить все поля по 1 разу и вернуться обратно?
		Классическаяф задача коммивояжера: Коммивояжер должен побывать в каждом из N городов по 1 разу. Зная расстояния между 2 любми городами, определить кратчайший маршрут.
		Переналадка станка: Станок выполняет N операций. После каждой требуется переналадка. Найти такой маршрут выполнения операций, чтобы суммарное время переналадки не превосходило заданного.

	Связный неориентированный граф - для двух любых вершин существует соединяющий их маршрут.
	Связный орграф - для любой пары вершин одна достижима для другой, или наоборот.
	Сильно связанный орграф - любые две вершины достижимы друг из друга.
	Слабо связанный орграф - не связный, но при замене всех дуг на ребра становится связным неориентированным

	Бикопонент (компонент сильной связности) - максимальный по включению сильносвязанный подграф графа.
	Граф можно рассматривать как набор бикомпонент, соединенный ребрами (не входящими в бикомпоненты)
	При этом все контуры входят в бикомпонент

	Если одно из воздействий попадает в вершину бикомпоненты, то оно беспрепятственно распостраняется во все другие ее вершины.
	Поэтому с т.з. прохождения инормации вся бикомпонента может рассматриваться как одна вершина.
	Такое преобразование называется граф Конденсации (Граф Герца)

	Сильно связанные компоненты являются особо слабыми местами в структуре этой системы
	Причины:
		Возмущение, попавшее в бикомпоненту, распостраняется во все ее вершины
		Может возникнуть эффект резонанса

	Такое зацикливание можно нейтрализовать - то есть преобразовать структуру системы в безконтурные графы
		Либо разорвать достаточное количество связей
		Либо повернуть направление нужного количества связей


Характеристические числа графа:
	
	Хроматическое число.
	Раскраска графа - разбиение множества его вершин на непересекающиеся подмножества, при этом каждое подмножества не содержит смежных вершин. Можно поставить в соответствие краски, при этом краски не будут наливаться друг на друга

	Хрматическое число графа - это наименьшее количество подмножеств, на которое можно разбить граф при раскраске
	Определение хроматического числа - задача, основанная на целочисленном линейном программировании.

	В компьютерных технологиях решается задача распределение регистров с т.з. экономии памяти


	Цикломатическое число.
	Наименьшее количество ребер, после удаления которых в графе не содержится ни одного цикла.
	Пример:
		Определение числа маршрутов при тестировании программы.

	Эксентриситет - это максимальное расстояние от данной вершины до других вершин.
	Он может счиаться по количеству вершин или с учетом веса.


Графовые модели в Computer Science
	
	Мы хотим преобразовать прогрмму/бизнес процесс/другое говно, описанное графом так, чтобы он или она выполнялась оптимальнее.
	Существует набор графовой модели программы
	Что можно рассматривать как вершины:
		Процедуры
		Циклы
		Итерации циклов
		Линейные участки
		Операции
		Срабатывания операторов
		etc

	Связь - отношение.
	В описании програм выделяют 2 типа отношений.
	Операционное отношение - Отношение по передаче управления (в программировании = сразу после)
	Информационное отношение - вершина Б использует в качестве аргумента некое значение, полученно в вершине А

	Из этих элементов можно построить различные графы програм.

	Граф управления программами: вершина - оператор. Связь - срабатывание оператора.
	И на программе в целом строится граф потоков управления - это представление всех путей, которые могут быть пройденны в процессе выполнения программы.

	Информационная модель - Возможность распараллеливания. похуй

	В любом случае, для оптимизаци программы выделяются циклы и компоненты сильной связности


Некоторые задачи теории графов и их применение в IT:
	Топологическая сортировка. 
	Вершины графа - цели. Пути - действия для выполнения цели. Топологическая сортировка - порядок, в котором могут быть достигнуты эти цели

	Нахождение наименьшего числа точек программы, в которые нужно поместить контроллеры для обнаружения ошибок циклических частей программы
	Эта задача соответствует наименьшего по мощности множества дуг, удаление которых разрывает все контуры, т.е. превращает орграф в безконтурный. Задача np-полная.
